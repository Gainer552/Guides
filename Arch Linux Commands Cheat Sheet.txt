**Arch Linux Commands Cheat Sheet**

General

#See your Operating System.
• cat /etc/os-release
#Show UEFI or BIOS.
• ls /sys/firmware/efi
Note: If nothing appears, then your system has a BIOS.
#Change hostname.
• echo ["NEW HOSTNAME"] > /etc/hostname
• sudo nano /etc/hosts
Note: Change it here as well to prevent DNS resolution errors.
• hostnamectl status
#See uses for commands and options.
• man [COMMAND]
Note: Press q to quit.
#Turn computer off.
• sudo poweroff.
#Reboot computer.
• sudo reboot
#Show currently logged in user.
• whoami
#Show all user activity on system.
• w
#Display how long machine has been running.
• uptime
#Change shell.
• chsh -s [SHELL NAME]
#Find the location of a package or binary.
• which [PACKAGE OR BINARY]
#Show all environment variables.
• env
#Add variable to PATH.
• export [IDENTIFIER]=[VALUE]
#Mount a filesystem to a specified mount point, an important operation for managing disks and partitions.
• mount [DEVICE] [/mnt/MOUNTPOINT]
Note: If no mount point exists create one with mkdir, in the /mnt dir.
#Update package checksums.
• updpkgsums
• makepkg -si

Time Synchronization

#Enable NTP.
• timedatectl set-ntp true.
#Verify change.
• timedatectl status
#Restart service.
• sudo systemctl restart systemd-timesyncd

Navigation

#Print working directory.
• pwd
#Change directory.
• cd [DIRECTORY]
#Go back to previous directory.
• cd .
#Navigate back 2 directories.
• cd ..

Searching

#List hidden/visible files and directories, with info.
• ls -lah
#Display how many items are in a directory.
• ls -la | wc -l
#Filter output of any command.
• [COMMAND] | grep ["KEYWORD"]
#Find any file, binary, package etc.
• sudo find / -name [FILENAME]
Note: You may also search for a specific package or binary with: sudo find / | grep ["BINARY OR PACKAGE"]
#Find all files with a specific extension.
• ls -lah | grep ["*.EXTENSION"]
#Search for specific environment variables.
• env | grep ["VARIABLE 1" "VARIABLE 2" "VARIABLE 3"]
#Find a specific directory.
• find -type d -name [DIRECTORY NAME]
#Find a specific file.
• find -type f -name [FILENAME]
#Find the path of a binary or package.
• which [BINARY OR PACKAGE]

File/directory creation, viewing, editing, moving, copying, renaming, and deletion.

#Create a file.
• touch [FILENAME AND EXTENSION]
#Redirect output of a command to a file.
• [COMMAND 1] [COMMAND 2] > [FILENAME AND EXTENSION]
#View contents of a file.
• cat [FILENAME]
#Edit a file.
• cat > [FILENAME]
Note: Enter text, then hit CTRL + C to save.
#Move a file.
• sudo mv [FILENAME] [EXACT PATH]
#Copy a file to a location.
• cp [FILENAME] [EXACT PATH]
Note: To copy to the same directory, exclude a path.
#Rename a file.
• sudo mv [FILENAME] [NEW FILENAME]
#Delete a file.
• rm [FILE1] [FILE2] [FILE3]
#Create a directory.
• mkdir [DIRECTORY NAME]
#Move a directory.
• sudo mv -R [DIRECTORY] [NEW DIRECTORY]
# Delete a directory.
• rm -r [DIRECTORY]

Error Handling

#Check the exit status of a command.
• [COMMAND] $?
#Do not show errors in the output.
• [COMMAND] 2 /dev/null

Nano Text Editor

#Install nano.
• sudo pacman -S nano
#Create a file with nano.
• nano [FILENAME AND EXTENSION]
#Save file.
• CTRL + X
• y + ENTER
#Edit a file.
• nano [FILENAME]
Notes: More keybinds are displayed, within the editor.

Aliases

#Create an alias.
• alias [NAME]=['COMMAND 1'] ['COMMAND 2'] ['COMMAND 3']
#Delete an alias.
• unalias [ALIAS]

Permission Management

#Change permissions.
• sudo chmod +[r, or w, or x, or rw, or rwx]
#Change permissions (octal notation).
• sudo chmod [NUMBERS]
#Change permissions for user only.
• sudo chmod u+[PERMISSIONS]
#Change permissions for group.
• sudo chmod g+[PERMISSIONS]
#Change permissions for a directory and sub-directories.
• sudo chmod -R [PERMISSIONS]

Changing ownership of files/directories.

#Change owner of a file.
• sudo chown [OWNER] [FILENAME]
#Change the owner of a directory and it's sub-directories.
• sudo chown -R [OWNER] [DIRECTORY]
Note: You can change the owner and group together with: sudo chown [OWNER]:[GROUP] [NAME]

#Stop a service.

• multi-user.target ()

SSH

#Install SSH.
• sudo pacman -S openssh
#Edit SSH config file.
• sudo nano /etc/ssh/sshd_config
Tips: Change port to a number in the thousands. Change PermitRootLogin to "no". Change MaxAuthTries to "3". Change MaxSessions to "1". Change PasswordAuthentication to "no". Generate encryption keys with: ssh-keygen -t ed25519.
#Enable SSH on startup.
• sudo systemctl enable --now sshd
#Start SSH service.
• sudo systemctl start sshd
#Restart SSH.
• sudo systemctl restart sshd
#Log into system.
• ssh [USERNAME]@[IP ADDRESS]
Note: Enter root as the user for unrestricted access.
#Logout of system.
• Logout

rsync

#Install rsync.
• sudo pacman -S rsync
#Copy files to your machine.
• rsync -v [FILENAME 1] [FILENAME 2] [FILENAME 3] [YOUR USERNAME]@[YOUR IP ADDRESS]:[YOUR DIRECTORY]
#Copy an entire directory with to your machine.
• rsync -rv [DIRECTORY] [YOUR USERNAME]@[YOUR IP ADDRESS]:[YOUR DIRECTORY]
Notes: If the host machine is running on a specific port, you must include the port like so: rsync -P [FILENAME] [YOUR USERNAME]@[YOUR IP ADDRESS]:[YOUR DIRECTORY]. You may also improve download speed, by compressing files with the -C option.

Permissions Reference (Octal Notation and Flags)

Flags:

• -r (Read)
• -w (Write)
• -x (Execute)

Octal Notation:

• 0	(No permissions).
• 1	(Execute only).
• 2	(Write only).
• 3	(Write and execute).
• 4	(Read only).
• 5	(Read and execute).
• 6	(Read and write).
• 7	Read, write, and execute (full access).

Error Codes Reference

1: EPERM - Operation not permitted. The process tried to perform an action without the necessary permissions (e.g., a non-root user trying to access a restricted file).
2: ENOENT - No such file or directory. The specified file or directory path does not exist.
3: ESRCH - No such process. The specified process ID (PID) does not exist.
4: EINTR - Interrupted system call. A system call was interrupted by a signal before it could complete.
5: EIO - I/O error. A physical input/output error occurred while communicating with a hardware device.
6: ENXIO - No such device or address. The device specified does not exist or the address is invalid.
11: EAGAIN (or EWOULDBLOCK) - Resource temporarily unavailable. The operation would block, and the resource is temporarily unavailable.
12: ENOMEM - Out of memory. The system could not allocate enough memory to complete the operation.
13: EACCES - Permission denied. Similar to EPERM, usually indicating a file access permission issue.
14: EFAULT - Bad address. The program provided an invalid memory address pointer.
16: EBUSY - Device or resource busy. An attempt was made to open a device or resource that was already in use.
17: EEXIST - File exists. An attempt was made to create a file that already exists.
22: EINVAL - Invalid argument. An invalid argument was passed to a system call or function.
28: ENOSPC - No space left on device. The disk partition is full.
30: EROFS - Read-only file system. An attempt was made to write to a file system that is mounted read-only.
